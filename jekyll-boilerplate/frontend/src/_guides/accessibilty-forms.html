Important

<!-- This must be present somewhere for it to be compliant-->
<p>Required fields are indicated with *</p>
<!-- If more than one form, make sure to use aria-labelledby -->
<form action="" data-role="form-wrapper" class="form">

Any descriptive text should be best outside of the form tag.

<p>Please fill in all of the fields of the form</p>
<form action="">
    <!-- Form content-->
</form>

Simple Input Group

This has most of the tags that will be needed, showing required fields and also showing an error message.

Some JS can be written so when the forms throw an error this adds an aria attribute to let users with screenreaders know there is an issue.

<div class="form__row">
      <label for="name">Name *</label>
      <input type="text" placeholder="Full name" id="name" required aria-describedby="rule error" />
      <span id="rule" style="padding: 5px; display: block;">The name must be over 3 characters long</span>
      <p id="error" role="alert" class="error">Please enter your name</p>
</div>

Adding required will use the JS validation of the field when checking the form. if you do not want this, then instead use: aria-required="true"

No labels

All inputs need a label to go with them, linking the for with the id of the field. If, for whatever reason there is no label then you would use arial-label.

<input type="text" name="name" placeholder="What's your name?" aria-label="What's your name?" required />

Another option would be to add a label, but hide it using a CSS class which is only for screen readers, like so:

<!-- Label hidden to user, still in for screen readers -->
<label for="name" class="visuallyhidden">What is your name? *</label>
<input type="text" id="name" placeholder="Full name"  required>

Select Dropdown

<div class="form__row">
        <label for="interestedIn">Interested in:</label>
         <select class="selectbox" id="interestedIn" data-placeholder="Please choose below">
           <!-- A blank option tag must be present to show the placeholder-->
           <option></option>
           <option value="1">1</option>
           <option value="2">2</option>
           <option value="3">3</option>
         </select>
      </div>

Radio/Checkboxes

<!-- Single Checkbox with Label -->
<div class="form__row">
        <div class="checkbox-wrap">
          <input id="issues" type="checkbox" value="no-issues">
          <label for="issues">There are no issues</label>
        </div>
</div>

To use multiple radio and checkboxes then please see below regarding using a fieldset.

Fieldset, Legend

You can split forms into sections, by using fieldset and then a legend to name the section, useful for screen readers. Do not use too much, only in obvious section when needed.

<form action="">
    <fieldset>
      <legend>Your Details</legend>
      <!-- Content-->
    </fieldset>
    <fieldset>
      <legend>Booking Details</legend>
      <!-- Content-->
    </fieldset>
  </form>

** Fieldset and legend should only be used when a higher-level label is necessary. Single checkboxes or basic radio buttons that make sense from their labels alone do not require fieldset and legend.

<div class="form__row">
        <fieldset>
          <legend>Payment Method</legend>
          <div class="radio-wrap">
            <input id="cash" type="radio" name="payment" value="cash" />
            <label for="cash"><span></span>Cash</label>
          </div>
          <div class="radio-wrap">
            <input id="check" type="radio" name="payment" value="check" />
            <label for="check"><span></span>Check</label>
          </div>
          <div class="radio-wrap">
            <input id="credit" type="radio" name="payment" value="credit" />
            <label for="credit"><span></span>Credit Card</label>
          </div>
        </fieldset>
      </div>
