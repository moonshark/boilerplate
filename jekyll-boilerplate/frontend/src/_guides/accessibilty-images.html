Accessibility with Images

Decide if decorative or functional. Whether you would be losing out if the image was not there.

functional - to serve a purpose - product image, shop images, logo, social icons

Decorative - a background, not needed, just to make it look nice.

This example shows which are decorative images:

NVDA for Windows and Mac has it's own VoiceOver as standard. Test the website using the keyboard only.

Hide from screen readers

To hide an image from screen readers, you can use aria-hidden="true". This is useful for icons that do not need to be read out by a screen reader.

Decorative Image that has no meaning

If it is a decorative image use alt="", as it tells the screen reader we know about it and that it is not important. No alt tag is NOT THE SAME. An empty tag is a better way of showing decorative

<img src="..." alt="">

Show to only screen readers

If you want to hide an image but only show it for screen readers, then you would just use the following CSS:

.visuallyhidden {
  border: 0;
  clip: rect(0 0 0 0);
  height: 1px;
  margin: -1px;
  overflow: hidden;
  padding: 0;
  position: absolute;
  width: 1px;
}

Background Images for Screen readers

If the image cannot be done using an image tag then a background image will have to do. You can do this like this:

<div class="hero" style="background-image: url('')" aria-label="Describe an image if needed"></div>

However, if there was some text within this div as well then a screen reader would read out the text but not the image description. So, you would need to add a separate span or div and make it like this:

<div class="hero">
  <h1>Visit Glasgow Airport</h1>
  <span style="background-image: url('')" aria-label="Describe an image if needed"></span>
</div>

Alt Text and what to include

A long Alt Text

If the image has a lot of text, then it is a good idea to add the full description, but not within the alt text. First state what the image is which is enough to understand -Native App Development, An infographic, and then get the rest of the text from the image, as shown below:

"Native App Development, An infographic. Native app development has three sections. 62% Usage: Android: Lorem ipsum dolor sit amet consectetur adipiscing elit. 33% Usage: iOS. Suspendissse maximus egestas metus ut vulputate. 5%. Usage: Windows. Donec lacinia augue vel pharetra dapibus.". 

<img src="..." alt="Native App Development, An infographic.">
<p class="sr-only">Native App Development, An infographic. Native app development has three sections. 62% Usage: Android: Lorem ipsum dolor sit amet consectetur adipiscing elit. 33% Usage: iOS. Suspendissse maximus egestas metus ut vulputate. 5%. Usage: Windows. Donec lacinia augue vel pharetra dapibus.</p>

This is fine, but not always will the content have the text below the image. A better way is like this:

<img src="..." alt="Native App Development, An infographic." aria-describedby="my-description">
<p id="my-description">Native App Development, An infographic. Native app development has three sections. 62% Usage: Android: Lorem ipsum dolor sit amet consectetur adipiscing elit. 33% Usage: iOS. Suspendissse maximus egestas metus ut vulputate. 5%. Usage: Windows. Donec lacinia augue vel pharetra dapibus.</p>
