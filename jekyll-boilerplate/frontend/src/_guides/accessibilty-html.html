Using role=""

HTML5 introduces many new elements, several of which correspond to, or at least are similar to the ARIA attributes.

You should not place the "role" attribute on another element that already has semantic meaning, like a <ul>. It should only be placed on <div> and <span> elements.

Which role do you use?

To find out which roles should go with which widget/tag then see this link, here. If you select, button for example then the two supported states are: aria-expanded and aria-pressed. You cannot make your own up.

Using Chrome Inspector to check

Select an element, like button or accordion item and then check 'Accessibility'. You should see the different aria states. If this is as expected then the aria is fine and working correctly.

HTML/ROLE alternative

You can see this table of semantics guide, listing all HTML elements available and links it to a role if available.

<!-- This is wrong, as nav is already a HTML 5 element -->
<nav role="navigation"></nav>

<!-- When the nav element has been used, you can use a div as only one nav on the page -->
<div class="nav" role="navigation">
  <ul>
    <li>Home</li>
    <li>Products</li>
    <li>Support</li>
  </ul>
</div>

The following table shows how ARIA landmarks should be applied to HTML5 Elements. No need for a role to be applied if using the correct tag

ARIA Landmark HTML5 Element Meaning
banner <header> contains the site-oriented content of each page, like the logo, usually at the top of the page. There should only be one banner landmark per page.
complementary <aside> a section of content that complements the main content but also retains its meaning when separated from the main content
contentinfo generic <div> acting as the footer contains content usually found in the footer of a page, like copyright and privacy statements. There should only be one contentinfo landmark per page.
form <form> or generic <div> contains form input elements which can be edited and submitted by the user
main <main> the main content of the page. There should only be one main landmark per page.
navigation <nav> a collection of navigation links to navigate the site or page
search <form> or generic <div> a search tool

Which tags to use and when?

You can see all of the aria practices here to see what tags etc need to be used for things like accordions, sliders etc.

HTML5 Element Meaning
<main> Main content of the page, this is what people came for
<section> Major sections of the page.
<article> Repeated content which can be used on their own. Such as posts, category snippets
<aside> Related to the main content, but not necessary to understand it, like sidebar
<nav> Any major navigation used for links

5 Rules of ARIA!

  2. 2- Don't change native semantics. Use the correct HTML - for example, do not use <a href="#" role="button">Menu</a>, when just using <button>Menu</button> will do.
  3. 3- All interactive ARIA roles need to be operable by keyboard.
  4. 4- Don't use role="presentation" or aria-hidden="true" on visible focusable elements as this will hide it from screen readers. Do not use on a visible focusable element.
  5. 5 - All interactive elements have an accessible name

Skip to main content

There is a skip to content option added for users to allow to skip the header and navigation for keyboard users so you would not need to hit tab to get past every link.

<a class="skip-main" href="#content">Skip to main content</a>

<!-- Then, have the ID going to the main tag -->
<main id="content">

Keyboard Users/Tabbing Order

Sometimes, the tab order may not work as you expect. There are ways around this by using tabindex="". The values can be -1, 0 or a number greater than 0.

Ideally it is best to put the HTML in the order you would want to tab. You could then use flexbox and other methods to change the order if needed.

<!-- The 2nd link will not be accessed by using tab -->
<ul>
  <li><a href="#">Here is a link</a></li>
  <li><a href="#" tabindex="-1">This will not be accessed via the tab</a></li>
  <li><a href="#">Here is a link</a></li>
</ul>
         
 <!-- If you want specifically a certain link or links then set the tab order to a number greater than 0 -->
<ul>
  <li><a href="#">Here is a link</a></li>
  <li><a href="#" tabindex="1">This link is first</a></li>
  <li><a href="#">Here is a link</a></li>
</ul>

Region Landmarks, Section's on the page

A region landmark is a perceivable section containing content that is relevant to a specific, author-specified purpose and sufficiently important that users will likely want to be able to navigate to the section easily and to have it listed in a summary of the page

Using section[aria-labelledby] attribute

This defines a label for each region using existing content on the page. The aria-labelledby relates to the content within the heading

<main>
  <h1>title for main content area<h1>

  <section aria-labelledby="region1">
    <h2 id="region1">About Us</h2>
    <!-- content for region area 1 -->
  </section>

  <section aria-labelledby="region2">>
    <h2 id="region2">Testimonials from clients</h2>
    <!-- content for region area 2 -->
  </section>

</main>

Using section[aria-label] attribute

Defines a label for each region using attribute value that is only visible to assistive technologies.

<main>
  <h1>title for main content<h1>

  <section aria-label="title for region area 1">
    <h2>title for region area 1</h2>
     <!-- content for region area 1 -->
  </section>

  <section aria-label="title for region area 2">
    <h2>title for region area 2</h2>
      <!-- content for region area 2 -->
  </section>
</main>

Multiple forms, asides, sections etc

If there are multiple section, aside, form tags on the page then each will need to have the aria-labelledby property added to it. If there is only one instance of it on each page, no label is needed

<!-- Only One aside does not need a role -->
<aside>
   <p>...</p>
</aside>
		
<!-- More than 1 complementary landmark need marking like this -->
<aside aria-labelledby="comp1">
	<p>...</p>
</aside>

<aside aria-labelledby="comp2">
	...
</aside>

Search

Use the search landmark instead of the form landmark when the form is used for search functionality.

<form role="search">
  <input type="search" aria-label="search text" size="20">
  <input type="submit" value="Search">
</form>
