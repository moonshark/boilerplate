Navigation

Only one nav element should be used per page. This should be the main navigation (unless the footer has identical links. No need to use navigation within the label, just a short description.

"The nav element represents a section of a page that links to other pages or to parts within the page: a section with navigation links. Not all groups of links on a page need to be in a nav element only sections that consist of major navigation blocks are appropriate for the nav element. In particular, it is common for footers to have a list of links to various key parts of a site, but the footer element is more appropriate in such cases, and no nav element is necessary for those links." - HTML5 Doctor
 <!-- A label can be added to describe it better -->
  <nav aria-label="Main">
      <ul>
        <!-- Content -->
      </ul>
    </nav>

<!-- For secondary navigation, use a div and add a role of navigation with a label, in this case it is in the footer.  -->
<div role="navigation" aria-label="Footer">
   <ul>
     <!-- Content -->
   </ul>
</div>

Identical Navigation Links

If a navigation landmark role or nav element in a document is repeated in a document, and both landmarks have identical content, use the same label for each landmark. An example of this would be repeating the main navigation at the top and bottom of the page.

<header>
  <nav aria-label="Main">
    <!-- list of links to main website locations -->
  </nav>
</header>
...
<footer>
  <nav aria-label="Main">
    <!-- list of links to main website locations -->
  </nav>
</footer>

Showing the user the current page for sighted and unsighted users

When you are on a page, normally we add a class like active to show the user the page they are on, however, screen readers cannot tell this. Simply add the following to a link and the screen reader will know.

<nav aria-label="Main">
  <ul>
    <li><a href="/products" aria-current="page" title="Our Products" class="active">Products</a></li>
    <li><a title="About Us" href="/about-us">About Us</a></li>
  </ul>
</nav>

The above works fine, but if you follow it, it will just refresh the page. You could replace the link to the ID taking you to the main part of the page

<nav aria-label="Main">
  <ul>
    <li><a href="/products" aria-current="page" title="Our Products" class="active">Products</a></li>
    <li><a title="About Us" href="#main-content">About Us</a></li>
  </ul>
</nav>

<main id="main-content">
  <!-- main content of the page -->
</main>
